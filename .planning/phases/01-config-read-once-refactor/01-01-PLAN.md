<objective>
Refactor internal/config/config.go so the YAML config file is read from disk exactly once per Load invocation. Currently the file is opened twice: first by v.ReadInConfig() internally via v.SetConfigFile, then explicitly by os.ReadFile for strict validation and rule parsing. This creates a race window and unnecessary complexity.

Purpose: Eliminate the duplicate OS read (CONF-01). The single rawYAML []byte feeds Viper, the KnownFields validator, and yaml.Unmarshal — all from one os.ReadFile call.
Output: Modified internal/config/config.go where v.SetConfigFile + v.ReadInConfig are replaced by os.ReadFile + v.SetConfigType("yaml") + v.ReadConfig(bytes.NewReader(rawYAML)).
</objective>

  <action>
In the Load function, replace the two-OS-read block (lines 113-128) with a single-read pattern.

Current code to remove/replace inside the `if configFile != ""` block:
```go
var rawYAML []byte

if configFile != "" {
    if _, err := os.Stat(configFile); err != nil {
        return nil, fmt.Errorf("config file %q not found: %w", configFile, err)
    }
    v.SetConfigFile(configFile)       // <-- REMOVE this line
    if err := v.ReadInConfig(); err != nil {  // <-- REMOVE this block
        return nil, fmt.Errorf("reading config file: %w", err)
    }
    var readErr error
    rawYAML, readErr = os.ReadFile(configFile)  // <-- CHANGE: move to be the primary read
    if readErr != nil {
        return nil, fmt.Errorf("re-reading config file for validation: %w", readErr)
    }
}
```

Replace with:
```go
var rawYAML []byte

if configFile != "" {
    if _, err := os.Stat(configFile); err != nil {
        return nil, fmt.Errorf("config file %q not found: %w", configFile, err)
    }
    var readErr error
    rawYAML, readErr = os.ReadFile(configFile)
    if readErr != nil {
        return nil, fmt.Errorf("reading config file: %w", readErr)
    }
    v.SetConfigType("yaml")
    if err := v.ReadConfig(bytes.NewReader(rawYAML)); err != nil {
        return nil, fmt.Errorf("reading config file: %w", err)
    }
}
```

Specific rules:
- Keep the `os.Stat` guard exactly as-is (preserves "config file %q not found" error message).
- Remove `v.SetConfigFile(configFile)` entirely — it is only used by ReadInConfig which is being dropped.
- Remove `v.ReadInConfig()` and its error block.
- Change `rawYAML, readErr = os.ReadFile(configFile)` to use `:=` short declaration if rawYAML is declared inside the block, OR keep the `var rawYAML []byte` at the top with `rawYAML, readErr = os.ReadFile(configFile)` inside the if-block (either form is fine; prefer the existing outer-declaration style to avoid restructuring the downstream code).
- Update the error message from "re-reading config file for validation" to "reading config file" (matches the Viper error message style already in use).
- Add `v.SetConfigType("yaml")` before `v.ReadConfig(...)` — this is REQUIRED. Without it Viper cannot determine the format and will fail (pitfall #1 in RESEARCH.md).
- Add `v.ReadConfig(bytes.NewReader(rawYAML))` after SetConfigType.
- Do NOT change the `yaml.NewDecoder(bytes.NewReader(rawYAML))` block below — it already correctly uses rawYAML.
- Do NOT change the `yaml.Unmarshal(rawYAML, &yc)` block below — it already correctly uses rawYAML.
- Keep the existing ordering: defaults → env setup (SetEnvPrefix, SetEnvKeyReplacer, AutomaticEnv) → SetConfigType → ReadConfig. Do not reorder these (pitfall #3 in RESEARCH.md).
- No import changes needed: bytes and os are already imported.
